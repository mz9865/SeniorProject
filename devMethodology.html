Development Methodology

Planned Activities and Artifacts

We are planning to have standups every other meeting. On top of this, we plan to have end of sprint ceremonies every 2 weeks to show off what work we’ve done to our customer and plan out our next sprint.

Tools

We will be using SmartSheets to keep track of our tasks and what’s the status of each. When testing, we will be using PyTest for our python code and Jest for our React code. For communication, we use Microsoft Teams and discord.

Metrics and Measurements

  • Product: Defect Density (Defect per LOC)
  • Product/Process: Volume of deliverables produced per unit effort (e.g. pages per hour, LOC per hour)
  • Process: Estimation Accuracy (% difference between estimated hours and actual hours)
  • Test:
    • Code Coverage
    • % Of tests passed successfully

Standards and Quality Practices

After work has been done for a task, the developer should keep their branch up to date and make sure no tests are failing. We’ll have a code review so someone other than the developer can have a look. After that, we’ll deploy the code after all tests are passed. Once deployed, the team will do further verification on the deployed environment and note any defects.
